# Zed Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a toggleable child-page mode that shows one heading-defined question and its answer at a time.

**Architecture:** Keep Notion fetching and block rendering in Server Components. Group heading sections on the server, render each question panel with the existing `NotionContent`, and pass those rendered panels into a small Client Component that owns toggle and Previous/Next state.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, Tailwind CSS, lucide-react, existing shadcn-style button component.

### Task 1: Centralize Zed Mode Grouping

**Files:**
- Create: `src/lib/zed-mode.ts`

- [ ] **Step 1: Add grouped question types and helpers**

Create `src/lib/zed-mode.ts` with:

```ts
import type {
  NotionContentBlock,
  NotionRichText,
  NotionTextBlock,
} from "@/types/notion";

export type ZedQuestionGroup = {
  question: NotionTextBlock;
  answerBlocks: NotionContentBlock[];
};

export function groupBlocksByHeading(
  blocks: NotionContentBlock[],
): ZedQuestionGroup[] {
  const groups: ZedQuestionGroup[] = [];
  let currentGroup: ZedQuestionGroup | null = null;

  for (const block of blocks) {
    if (isHeadingBlock(block)) {
      currentGroup = {
        question: block,
        answerBlocks: [],
      };
      groups.push(currentGroup);
      continue;
    }

    if (currentGroup) {
      currentGroup.answerBlocks.push(block);
    }
  }

  return groups;
}

export function plainRichText(text: NotionRichText[]): string {
  return text.map((item) => item.plainText).join("");
}

export function isHeadingBlock(
  block: NotionContentBlock,
): block is NotionTextBlock {
  return (
    block.type === "heading_1" ||
    block.type === "heading_2" ||
    block.type === "heading_3"
  );
}
```
